## 1. 项目概述

输入羽毛球运动视频，输出：
- 带骨骼+球轨迹标注的视频
- 运动参数（关节角度、速度、倾斜）
- 问题/精彩片段慢镜头
- LLM教练建议

## 3. 核心模块

### 3.1 姿态估计 (pose_estimator.py)

**RTMPose + 运动员过滤**

```
帧 → RTMPose检测所有人 → 球场检测 → 运动员评分 → 选最佳1人
```

评分维度：
- 球拍检测 50%（手腕区域形状分析）
- 身高占比 20%（站着>坐着）
- 球场内 15%（HSV颜色分割）
- 位置 10%（画面中央）
- 面积 5%

### 3.2 球追踪 (ball_tracker.py)

**TrackNet深度学习模型**

```
3帧 → 拼接为9通道 → TrackNet → 256通道热力图 → HoughCircles → 球坐标
```

- 输入: (1, 9, 360, 640)
- 输出: 球坐标 (x, y)
- 支持PyTorch/ONNX双后端
- GPU推理: ~0.02秒/帧

### 3.3 事件检测 (event_detector.py)

**五层检测架构**

```
帧数据 → 滑窗聚合(15帧) → 活动状态机 → 挥拍检测 → 问题检测 → 事件合并
```

| 层 | 技术 | 作用 |
|---|------|------|
| 滑窗聚合 | 15帧窗口/5帧步长 | 消除单帧抖动 |
| 状态机 | PLAYING/MOVING/RESTING | 只在打球时检测 |
| 挥拍检测 | 速度峰值+加速度突变 | 标记挥拍时刻 |
| 问题检测 | 持续6帧+10°容差 | 过滤瞬间异常 |
| 合并 | 同类事件90帧内合并 | 避免重复 |

### 3.4 LLM教练 (llm/)

**Skill Router模式**

```python
# 不同功能用不同Prompt
skills = {
    "technique_diagnosis": "诊断动作问题",
    "training_plan": "制定训练计划",
    "risk_alert": "评估伤病风险",
}
```

输入: 结构化JSON（关节角度/速度/事件）
输出: JSON格式建议（summary/problems/training_tips）

## 6. 视频要求

| 要求 | 原因 |
|------|------|
| 近距离拍摄（运动员>25%画面） | RTMPose需要足够大的人体 |
| 固定机位 | 追踪稳定性 |
| 光线充足 | 关键点检测精度 |
| 不适合广播远景 | 运动员太小，骨架不准 |

## 7. 性能基准

| 设备 | 骨骼(300帧) | 球追踪(300帧) | 总计 |
|------|------------|--------------|------|
| CPU | ~15秒 | ~750秒 | ~13分钟 |
| GPU (CUDA) | ~3秒 | ~6秒 | ~9秒 |
| GPU (TensorRT) | ~2秒 | ~3秒 | ~5秒 |

## 8. 已知限制

| 限制 | 说明 | 解决方案 |
|------|------|---------|
| 广播远景不准 | 运动员太小 | 使用近距离视频 |
| CPU球追踪慢 | TrackNet 2.5秒/帧 | 使用GPU |
| 球检测率37% | 640×360分辨率球太小 | 更高分辨率视频 |
| 单人追踪 | 当前只追踪1人 | 后续支持双打 |
